# ADR-002: Respawn + Checkpoint แทน permanent Game Over

## Context

เกมเดิม (endless runner แบบดั้งเดิม) ให้ "ตายแล้วจบเกม" ทันที แต่ Penguin Dash ออกแบบมาให้เล่นพร้อมกันทั้งห้องเรียนแบบ Kahoot — ถ้าใครพลาดตั้งแต่ช่วงต้นแล้วต้องออกจากเกมเลย คนนั้นจะนั่งรอเฉย ๆ จนจบ session รู้สึกเสียโอกาสและไม่มีส่วนร่วม โดยเฉพาะในบริบทห้องเรียนที่ต้องการให้ทุกคนได้เรียนรู้ ไม่ใช่แค่คนที่เล่นเก่ง

## Decision

- ชนสิ่งกีดขวาง → เสีย HP (หรือ Heat Meter) ไม่ใช่ตายทันที
- HP หมด → เข้าสถานะ **Respawn**: รอ 3 วินาที + คะแนนลด 10% → เกิดใหม่ที่ **checkpoint ล่าสุด** (ทุก 100 ม.) → เล่นต่อ คะแนนสะสมต่อได้ ไม่รีเซ็ต
- นับ `respawn_count` เก็บลง RunRecord เป็น event — ใช้แสดงบน dashboard เป็นตัวชี้วัดความพยายาม/ความสม่ำเสมอ ไม่ใช่แค่คะแนนดิบ
- ธีมในเกม: โดรน P.E.N.G.U.I.N "ถูกซ่อม/รีชาร์จ" แล้วกลับมาปฏิบัติภารกิจ — penalty สอดคล้องกับเนื้อเรื่อง ไม่ใช่กลไกแปะทับ
- เกมยังมีจุดจบจริง (ไม่ใช่ endless ตลอดไป): เส้นชัย 1,000 ม. → Boss → Mission Complete

## Consequences

- ได้: ทุกคนเล่นจนจบ session ได้จริง, ยังมีแรงกดดันจาก penalty (เวลา+คะแนน) แต่ไม่รุนแรงจนหมดสนุก, การแข่งขันยังยุติธรรม (คนเล่นดีคะแนนสูงกว่า แต่คนพลาดไต่กลับได้), ครูเห็นความคืบหน้าของทุกคนแบบ real-time ตลอด session
- เสีย: ต้องมี state เพิ่ม (checkpoint position, respawn_count) และ UI overlay "Respawning..." ระหว่างรอ — งานเพิ่มใน D2 แต่ไม่ซับซ้อนมาก
- Trade-off ที่ยอมรับ: เกมไม่มี "ตายจริง" อีกต่อไป ผู้เล่นเก่งมากอาจรู้สึกว่า penalty เบาไป — ยอมรับได้เพราะเป้าหมายหลักคือการเรียนรู้ทั้งห้อง ไม่ใช่ competitive ranking ที่โหดที่สุด
